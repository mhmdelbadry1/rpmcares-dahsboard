Introduction

This document covers the Tenovi API as used by the RPMCares client account. It was produced by reverse-engineering the app.tenovi.com web application's network traffic and verified against the live API.

⚠️ SENSITIVE — INTERNAL USE ONLY. This document contains live API credentials, a TOTP secret, and account data. Do not share externally or commit to version control.

Two Separate APIs

Tenovi exposes two completely different APIs under the same domain. They are not interchangeable:

APIBase PathAuth TypeWhat it contains
RPM / Clinical https://api2.tenovi.com/clients/rpmcares/ Token (session) Facilities, patient rosters, measurements, clinical modules (RPM/RTM)
HWI / Hardware https://api2.tenovi.com/clients/rpmcares/hwi/ Api-Key Gateways, hardware devices, bulk orders — no clinical patient data
ℹ️ The RPM API is the one that powers the web app and contains all clinic/patient data. The HWI API has an official Swagger spec at app.tenovi.com/hwi/schema but the RPM API has no public docs — this document is the only reference.

Authentication Flow

The RPM API uses a two-step login. The same endpoint /auth/ is called twice: first with username/password, then again with the TOTP code added. The token returned in step 2 is used for all subsequent calls.

⚠️ Token rotation: A new token is issued on every login. The previous token is immediately invalidated. This means if two processes log in concurrently, one will kill the other's token.
1
Compute TOTP code — use the TOTP secret to generate the current 6-digit code (valid for 30 seconds). See TOTP Automation section.
2
POST /auth/ with {username, password} only → returns mode: "setup_otp" and a session_cookie UUID. (This step can be skipped — you may generate your own UUID for session_cookie.)
3
POST /auth/ again with {username, password, otp, otp_method: "A", session_cookie} → returns {"token": "...", "status": "Logged in"}.
4
Use token in every subsequent request: Authorization: Token <token>.

TOTP Automation

The authenticator OTP is computed locally using the shared secret — no network call, no waiting. The algorithm is RFC 6238 TOTP (same as Google Authenticator). It uses the current Unix time divided into 30-second windows.

# Requires only Python standard library
import hmac, hashlib, time, base64, uuid, urllib.request, json

TOTP_SECRET  = "EFTW5B2MWJSAV4Z3ZL2DVEYPUWQLU4VV"
USERNAME     = "hyousif@rpmcares.com"
PASSWORD     = "Storm@123"
BASE_URL     = "https://api2.tenovi.com/clients/rpmcares"

def compute_totp(secret):
    key = base64.b32decode(secret.upper())
    counter = int(time.time() // 30)
    h = hmac.new(key, counter.to_bytes(8, 'big'), hashlib.sha1).digest()
    offset = h[-1] & 0xf
    code = (int.from_bytes(h[offset:offset+4], 'big') & 0x7fffffff) % 1000000
    return f"{code:06d}"

def get_token():
    otp = compute_totp(TOTP_SECRET)
    payload = {
        "username": USERNAME,
        "password": PASSWORD,
        "otp": otp,
        "otp_method": "A",
        "session_cookie": str(uuid.uuid4())
    }
    data = json.dumps(payload).encode()
    req = urllib.request.Request(
        "https://api2.tenovi.com/auth/", data=data, method="POST",
        headers={"Content-Type": "application/json", "Origin": "https://app.tenovi.com"}
    )
    resp = json.load(urllib.request.urlopen(req))
    return resp["token"]

const TOTP_SECRET = "EFTW5B2MWJSAV4Z3ZL2DVEYPUWQLU4VV";
const USERNAME    = "hyousif@rpmcares.com";

function base32Decode(s) {
  const alphabet = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ234567';
  let bits = '';
  for (const c of s.replace(/=+$/, '').toUpperCase()) {
    const v = alphabet.indexOf(c);
    if (v === -1) continue;
    bits += v.toString(2).padStart(5, '0');
  }
  const bytes = [];
  for (let i = 0; i + 8 <= bits.length; i += 8)
    bytes.push(parseInt(bits.substring(i, i + 8), 2));
  return Buffer.from(bytes);
}

function computeTOTP(secret) {
  const key = base32Decode(secret);
  const counter = Math.floor(Date.now() / 1000 / 30);
  const buf = Buffer.alloc(8);
  buf.writeBigInt64BE(BigInt(counter));
  const h = crypto.createHmac('sha1', key).update(buf).digest();
  const offset = h[h.length - 1] & 0xf;
  const code = ((h[offset] & 0x7f) << 24 | h[offset+1] << 16 | h[offset+2] << 8 | h[offset+3]) % 1000000;
  return code.toString().padStart(6, '0');
}

async function getToken() {
  const res = await fetch('https://api2.tenovi.com/auth/', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', 'Origin': 'https://app.tenovi.com' },
    body: JSON.stringify({
      username: USERNAME, password: PASSWORD,
      otp: computeTOTP(TOTP_SECRET), otp_method: 'A',
      session_cookie: crypto.randomUUID()
    })
  });
  const data = await res.json();
  return data.token;
}

Python — Get all active patients across all facilities

import urllib.request, json

def api_get(url, token):
    req = urllib.request.Request(url, headers={
        "Authorization": f"Token {token}", "Accept": "application/json"
    })
    return json.load(urllib.request.urlopen(req))

token = get_token()
facilities = api_get("https://api2.tenovi.com/clients/rpmcares/facilities/", token)

all_patients = []
for facility in facilities:
    fid   = facility["id"]
    fname = facility["name"]
    url   = f"https://api2.tenovi.com/clients/rpmcares/rpm/facilities/{fid}/patients/?status=AC&page_size=500"
    data  = api_get(url, token)
    for p in data["results"]:
        p["clinic_name"] = fname     # attach clinic name
        all_patients.append(p)

print(f"Total active patients: {len(all_patients)}")
for p in all_patients:
    print(f"  [{p['clinic_name']}] {p['patient']['name']} — {p['patient']['devices'][0]['module'] if p['patient']['devices'] else 'No device'}")
