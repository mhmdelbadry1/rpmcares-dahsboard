Introduction

This document covers the Tenovi API as used by the RPMCares client account. It was produced by reverse-engineering the app.tenovi.com web application's network traffic and verified against the live API.

⚠️ SENSITIVE — INTERNAL USE ONLY. This document contains live API credentials, a TOTP secret, and account data. Do not share externally or commit to version control.

Two Separate APIs

Tenovi exposes two completely different APIs under the same domain. They are not interchangeable:

APIBase PathAuth TypeWhat it contains
RPM / Clinical https://api2.tenovi.com/clients/rpmcares/ Token (session) Facilities, patient rosters, measurements, clinical modules (RPM/RTM)
HWI / Hardware https://api2.tenovi.com/clients/rpmcares/hwi/ Api-Key Gateways, hardware devices, bulk orders — no clinical patient data
ℹ️ The RPM API powers the web app and contains all clinic, patient, and measurement data. The HWI API has no measurement endpoints — the /hwi/schema/swagger-ui/ path is not accessible via the API; it is a frontend SPA route on app.tenovi.com only. This document is the only reference for the RPM API.
⚠️ Do NOT use the HWI API to fetch patient measurements. HWI patient IDs (16 records only) are completely separate from RPM patient UUIDs. Sending RPM UUIDs to any HWI endpoint returns 404 or empty data. Patient measurement history is only available via the RPM endpoint GET /data/patients/{uuid}/measurements/.

Authentication Flow

The RPM API uses a two-step login. The same endpoint /auth/ is called twice: first with username/password, then again with the TOTP code added. The token returned in step 2 is used for all subsequent calls.

⚠️ Token rotation: A new token is issued on every login. The previous token is immediately invalidated. This means if two processes log in concurrently, one will kill the other's token.
1
Compute TOTP code — use the TOTP secret to generate the current 6-digit code (valid for 30 seconds). See TOTP Automation section.
2
POST /auth/ with {username, password} only → returns mode: "setup_otp" and a session_cookie UUID. (This step can be skipped — you may generate your own UUID for session_cookie.)
3
POST /auth/ again with {username, password, otp, otp_method: "A", session_cookie} → returns {"token": "...", "status": "Logged in"}.
4
Use token in every subsequent request: Authorization: Token <token>.

TOTP Automation

The authenticator OTP is computed locally using the shared secret — no network call, no waiting. The algorithm is RFC 6238 TOTP (same as Google Authenticator). It uses the current Unix time divided into 30-second windows.

# Requires only Python standard library
import hmac, hashlib, time, base64, uuid, urllib.request, json

TOTP_SECRET  = "EFTW5B2MWJSAV4Z3ZL2DVEYPUWQLU4VV"
USERNAME     = "hyousif@rpmcares.com"
PASSWORD     = "Storm@123"
BASE_URL     = "https://api2.tenovi.com/clients/rpmcares"

def compute_totp(secret):
    key = base64.b32decode(secret.upper())
    counter = int(time.time() // 30)
    h = hmac.new(key, counter.to_bytes(8, 'big'), hashlib.sha1).digest()
    offset = h[-1] & 0xf
    code = (int.from_bytes(h[offset:offset+4], 'big') & 0x7fffffff) % 1000000
    return f"{code:06d}"

def get_token():
    otp = compute_totp(TOTP_SECRET)
    payload = {
        "username": USERNAME,
        "password": PASSWORD,
        "otp": otp,
        "otp_method": "A",
        "session_cookie": str(uuid.uuid4())
    }
    data = json.dumps(payload).encode()
    req = urllib.request.Request(
        "https://api2.tenovi.com/auth/", data=data, method="POST",
        headers={"Content-Type": "application/json", "Origin": "https://app.tenovi.com"}
    )
    resp = json.load(urllib.request.urlopen(req))
    return resp["token"]

const USERNAME    = "hyousif@rpmcares.com";

function base32Decode(s) {
  const alphabet = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ234567';
  let bits = '';
  for (const c of s.replace(/=+$/, '').toUpperCase()) {
    const v = alphabet.indexOf(c);
    if (v === -1) continue;
    bits += v.toString(2).padStart(5, '0');
  }
  const bytes = [];
  for (let i = 0; i + 8 <= bits.length; i += 8)
    bytes.push(parseInt(bits.substring(i, i + 8), 2));
  return Buffer.from(bytes);
}

function computeTOTP(secret) {
  const key = base32Decode(secret);
  const counter = Math.floor(Date.now() / 1000 / 30);
  const buf = Buffer.alloc(8);
  buf.writeBigInt64BE(BigInt(counter));
  const h = crypto.createHmac('sha1', key).update(buf).digest();
  const offset = h[h.length - 1] & 0xf;
  const code = ((h[offset] & 0x7f) << 24 | h[offset+1] << 16 | h[offset+2] << 8 | h[offset+3]) % 1000000;
  return code.toString().padStart(6, '0');
}

async function getToken() {
  const res = await fetch('https://api2.tenovi.com/auth/', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', 'Origin': 'https://app.tenovi.com' },
    body: JSON.stringify({
      username: USERNAME, password: PASSWORD,
      otp: computeTOTP(TOTP_SECRET), otp_method: 'A',
      session_cookie: crypto.randomUUID()
    })
  });
  const data = await res.json();
  return data.token;
}

Python — Get all active patients across all facilities

import urllib.request, json

def api_get(url, token):
    req = urllib.request.Request(url, headers={
        "Authorization": f"Token {token}", "Accept": "application/json"
    })
    return json.load(urllib.request.urlopen(req))

token = get_token()
facilities = api_get("https://api2.tenovi.com/clients/rpmcares/facilities/", token)

all_patients = []
for facility in facilities:
    fid   = facility["id"]
    fname = facility["name"]
    url   = f"https://api2.tenovi.com/clients/rpmcares/rpm/facilities/{fid}/patients/?status=AC&page_size=500"
    data  = api_get(url, token)
    for p in data["results"]:
        p["clinic_name"] = fname     # attach clinic name
        all_patients.append(p)

print(f"Total active patients: {len(all_patients)}")
for p in all_patients:
    print(f"  [{p['clinic_name']}] {p['patient']['name']} — {p['patient']['devices'][0]['module'] if p['patient']['devices'] else 'No device'}")

// Paste as section 2 of Fetch Readings node (keep SmartMeter section 1 unchanged)
const TENOVI_BASE        = 'https://api2.tenovi.com/clients/rpmcares';
const TENOVI_METRIC_BP   = 'cf2166eb-9e8f-4445-ac6d-bbc6abd85e87'; // blood_pressure
const TENOVI_METRIC_GLUC = '0375cc1b-9e67-49f3-952e-a5f51a88d1db'; // glucose

const TOKEN = $('Tenovi Login').first().json.tenoviToken;
if (!TOKEN) throw new Error('No token from Tenovi Login node');
const authHeaders = { 'Authorization': `Token ${TOKEN}`, 'Accept': 'application/json' };

const tenoviPatients = $('Fetch All Patients').all()
  .map(it => it.json)
  .filter(p => p && p.__source === 'tenovi' && p.patientId);

for (const patient of tenoviPatients) {
  for (const metricId of [TENOVI_METRIC_BP, TENOVI_METRIC_GLUC]) {
    let nextUrl = `${TENOVI_BASE}/data/patients/${patient.patientId}/measurements/` +
      `?metric__id=${metricId}&timestamp__gte=${dateStart}&timestamp__lt=${dateEnd}&page_size=100`;
    while (nextUrl) {
      const resp = await this.helpers.httpRequest({ method: 'GET', url: nextUrl, headers: authHeaders });
      const pData = typeof resp === 'string' ? JSON.parse(resp) : resp;
      for (const m of pData.results || []) {
        const name = m.measurement_metric?.name || '';
        const v1 = parseFloat(m.value), v2 = parseFloat(m.secondary_value);
        const base = { patient_id: patient.patientId, timestamp: m.timestamp,
          date_recorded: m.timestamp, reading_id: `tenovi_${patient.patientId}_${m.timestamp}_${name}`,
          __clinicName: patient.groupName, __source: 'tenovi' };
        if (name === 'blood_pressure')
          allRecords.push({ ...base, device_type:'blood_pressure', reading_type:'blood_pressure',
            systolic: v1, systolic_mmhg: v1, diastolic: v2, diastolic_mmhg: v2 });
        else if (name === 'glucose')
          allRecords.push({ ...base, device_type:'blood_glucose', reading_type:'blood_glucose',
            measurement_type:'RBS', glucose: v1, blood_glucose_mgdl: v1, before_meal: false });
      }
      nextUrl = (pData.next && pData.results?.length === 100) ? pData.next : null;
    }
  }
}

n8n Workflow Blueprint

Current production architecture (rpmcares RPM Agent). Tenovi runs in parallel with SmartMeter — all nodes handle both sources.

Phase 1 — Login & Patients

1
Tenovi Login (Code Node)
Pure-JS TOTP (no require('crypto') — n8n sandbox blocks it). Computes OTP → POSTs to /auth/ → outputs { tenoviToken }. Reusable — referenced by downstream nodes via $('Tenovi Login').first().json.tenoviToken.
2
Fetch All Patients (Code Node, executeOnce: true)
Section 1: SmartMeter per-clinic pagination. Section 2: Tenovi — GET /facilities/ → for each facility GET /rpm/facilities/{id}/patients/?status=AC&page_size=500 with next-based pagination. Reads token from Tenovi Login node. Outputs unified shape: { patientId, patientName, groupName, phone, status, __source }. Phones normalized to E.164 (+1XXXXXXXXXX).

Phase 2 — Readings

3
Fetch Readings & Alerts (Code Node)
Section 1: SmartMeter — POST /api/readings per clinic (blood_glucose, blood_pressure). Section 2: Tenovi — for each patient with __source==='tenovi', two calls to GET /data/patients/{uuid}/measurements/: once with metric__id=cf2166eb... (blood_pressure) and once with metric__id=0375cc1b... (glucose). Uses RPM Token auth. Output shape matches SmartMeter: { patient_id, systolic, systolic_mmhg, diastolic, diastolic_mmhg, timestamp, reading_id, __clinicName, __source } for BP; same pattern for glucose.

Key implementation notes

TopicDetail
TOTP in n8nrequire('crypto') is silently blocked in n8n Code Node sandbox — use the pure-JS SHA1/HMAC-SHA1 implementation from the TOTP section above.
PaginationBoth patients and measurements use next field (absolute URL). Stop when pData.next is null — do NOT stop on results.length < page_size (server ignores large page_size for patients).
Token invalidationEach new login invalidates all prior tokens. Only one login per workflow run. Do not log in inside a loop.
Patient IDsTenovi RPM patients have UUID IDs (74db2a23-...). These are the canonical IDs for all RPM API calls. HWI patient IDs (16 records) are unrelated — never mix them.
Phone normalizationTenovi returns phones as 469-230-6753 (no country code). Normalize: strip non-digits → if 10 digits prepend +1.
Measurement reading_idTenovi measurements have a UUID id field — use it directly as reading_id for dedup in AuditLog.
✅ With both phases complete: ~2086 total patients (SmartMeter + Tenovi), readings for all active devices, anomaly detection and missing-reading reminders work for both sources with the same downstream nodes.
